เรื่อง: ฟังก์ชันเพิ่มเติมสำหรับโปรไฟล์ผู้ใช้งานและการยืนยันตัวตน
เพื่อให้แอปพลิเคชัน Jobs4Dent มีความน่าเชื่อถือและใช้งานได้จริงสำหรับทุกฝ่าย ขอให้ทีมพัฒนาเพิ่ม 3 ฟีเจอร์หลักดังต่อไปนี้:
1. Feature: โปรไฟล์ย่อของทันตแพทย์ (Dentist's Mini-Resume)
* User Role: ทันตแพทย์ (Dentist)
* Objective: สร้างโปรไฟล์ฉบับย่อที่สรุปข้อมูลสำคัญของทันตแพทย์ ทำให้คลินิกสามารถประเมินคุณสมบัติเบื้องต้นได้อย่างรวดเร็ว และใช้เป็นข้อมูลหลักสำหรับฟีเจอร์ "ส่ง Resume อัตโนมัติ" เมื่อทันตแพทย์กดสมัครงาน
Key Components / User Flow:
1. ในหน้าแก้ไขโปรไฟล์ (Edit Profile) ของทันตแพทย์ ให้เพิ่มส่วน "ข้อมูลสรุปสำหรับสมัครงาน"
2. ผู้ใช้ (ทันตแพทย์) จะต้องกรอกข้อมูลในช่องที่กำหนดให้ครบถ้วน
3. ข้อมูลนี้จะถูกแสดงผลเป็นส่วนแรกๆ หรือเป็น Pop-up เมื่อคลินิกเปิดดูรายชื่อผู้สมัคร
Data Fields ที่ต้องจัดเก็บ:
* สถาบันที่สำเร็จการศึกษา (Education Institute):
   * Type: String
   * Description: ชื่อมหาวิทยาลัย/สถาบันที่จบการศึกษาหลักสูตรทันตแพทยศาสตรบัณฑิต
   * UI Suggestion: Text Input Field
* ประสบการณ์ทำงาน (Years of Experience):
   * Type: Number
   * Description: จำนวนปีของประสบการณ์ทำงานหลังจบการศึกษา
   * UI Suggestion: Number Input หรือ Dropdown (เช่น 0-1 ปี, 2-3 ปี, 4-5 ปี, 5+ ปี)
* ความสามารถ/หัตถการที่ทำได้ (Core Competencies):
   * Type: Array of Strings
   * Description: รายการหัตถการที่ทำได้และมีความเชี่ยวชาญ
   * UI Suggestion: ระบบ Tag Input หรือ Checkboxes ที่มีรายการมาตรฐานให้เลือก (เช่น ทันตกรรมทั่วไป, รักษารากฟัน, ศัลยกรรมช่องปาก, จัดฟัน, รากฟันเทียม) และมีช่อง "อื่นๆ" ให้ผู้ใช้พิมพ์เพิ่มเองได้
* หัตถการที่ไม่สะดวกทำ (Work Limitations/Preferences):
   * Type: Array of Strings
   * Description: รายการหัตถการที่ผู้ใช้ไม่สะดวกใจที่จะทำ เพื่อช่วยให้คลินิกคัดกรองได้ง่ายขึ้น
   * UI Suggestion: ระบบ Tag Input หรือ Checkboxes เช่นเดียวกับด้านบน
Acceptance Criteria:
* [ ] ทันตแพทย์สามารถกรอก, แก้ไข และบันทึกข้อมูลทั้ง 4 ส่วนนี้ในหน้าโปรไฟล์ของตนเองได้
* [ ] เมื่อทันตแพทย์กดสมัครงาน ข้อมูลส่วนนี้จะต้องถูกส่งไปให้คลินิกเห็นเป็นลำดับแรก
* [ ] ระบบสามารถกรอง/ค้นหาทันตแพทย์จากข้อมูล "ความสามารถ/หัตถการที่ทำได้" ได้
